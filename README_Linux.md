## Prerequisites

### Required Software
- **GCC/G++ Compiler**: C++ compiler toolchain
  - Installation: `sudo apt update && sudo apt install build-essential`
  - Compiler path: `/usr/bin/g++`
- **Git** (Optional but recommended): Version control
  - Installation: `sudo apt install git`
- **Visual Studio Code**: Code editor
  - Download from: [VS Code Official Website](https://code.visualstudio.com/)
  - Or install via Snap: `sudo snap install --classic code`
  - Or install via APT: 
    ```bash
    wget -qO- https://packages.microsoft.com/keys/microsoft.asc | gpg --dearmor > packages.microsoft.gpg
    sudo install -o root -g root -m 644 packages.microsoft.gpg /etc/apt/trusted.gpg.d/
    sudo sh -c 'echo "deb [arch=amd64,arm64,armhf signed-by=/etc/apt/trusted.gpg.d/packages.microsoft.gpg] https://packages.microsoft.com/repos/code stable main" > /etc/apt/sources.list.d/vscode.list'
    sudo apt update
    sudo apt install code
    ```

3. **C/C++ Themes** (`ms-vscode.cpptools-themes`)
   - Enhanced syntax highlighting themes for C++

4. **CMake Tools** (`ms-vscode.cmake-tools`) - Optional
   - For projects using CMake build system

## Building and Running

### Method 1: Using VS Code Build Task (Recommended)

**Note**: You'll need to create a Linux-specific `.vscode/tasks.json` file or modify the existing one:

1. Create or modify `.vscode/tasks.json`:

```json
{
    "version": "2.0.0",
    "tasks": [
        {
            "type": "cppbuild",
            "label": "C/C++: g++ build active file",
            "command": "/usr/bin/g++",
            "args": [
                "-fdiagnostics-color=always",
                "-g",
                "${file}",
                "-o",
                "${fileDirname}/${fileBasenameNoExtension}"
            ],
            "options": {
                "cwd": "${fileDirname}"
            },
            "group": {
                "kind": "build",
                "isDefault": true
            },
            "detail": "Task generated by Debugger for Ubuntu Linux."
        }
    ]
}
```

2. Open the project in VS Code
3. Open the C++ file you want to compile (e.g., `main.cpp`)
4. Use the build task:
   - Press `Ctrl+Shift+P` to open Command Palette
   - Type "Tasks: Run Build Task" and select it
   - Choose "C/C++: g++ build active file"

### Method 2: Using Terminal

Open the integrated terminal in VS Code (`Ctrl+`` ` `) and run:

```bash
# Compile main.cpp or the corresponding name of the cpp file
g++ -fdiagnostics-color=always -g main.cpp -o main

# Run the executable
./main
```

```bash
# Compile VoltageCalc.cpp
g++ -fdiagnostics-color=always -g VoltageCalc.cpp -o VoltageCalc

# Run the executable
./VoltageCalc
```

### Method 3: Using Makefile (Advanced)

Create a simple Makefile for more complex builds:

```makefile
# Makefile
CXX = g++
CXXFLAGS = -Wall -Wextra -std=c++17 -g
TARGET = main
SOURCES = main.cpp

$(TARGET): $(SOURCES)
	$(CXX) $(CXXFLAGS) -o $(TARGET) $(SOURCES)

clean:
	rm -f $(TARGET)

.PHONY: clean
```

Then build with:
```bash
make          # Build the project
make clean    # Clean built files
```

## Configuration

### Linux-Specific Configuration

- **Compiler**: `/usr/bin/g++` (default)
- **Flags**: 
  - `-fdiagnostics-color=always`: Colored compiler output
  - `-g`: Include debugging information
  - `-Wall`: Enable most warning messages
  - `-Wextra`: Enable extra warning messages
  - `-std=c++17`: Use C++17 standard (adjust as needed)

### File Permissions

On Linux, compiled executables don't have `.exe` extensions and may need execute permissions:

```bash
# Make executable (usually automatic)
chmod +x main

# Run executable
./main
```

## Troubleshooting

### Common Issues

1. **Compiler not found**
   - Install build-essential: `sudo apt update && sudo apt install build-essential`
   - Verify installation: `g++ --version`
   - Check the compiler path in `.vscode/tasks.json`

2. **VS Code doesn't recognize C++ syntax**
   - Install the C/C++ extension pack
   - Reload VS Code window (`Ctrl+Shift+R`)

3. **Build task not available**
   - Ensure you have a C++ file open in the editor
   - Check that `.vscode/tasks.json` exists and is properly configured for Linux
   - Create a Linux-specific tasks.json file

4. **Permission denied when running executable**
   - Make sure the file has execute permissions: `chmod +x filename`

5. **Code command not found**
   - If installed via Snap: `sudo snap alias code.code code`
   - If installed manually: Add VS Code to PATH or reinstall using package manager

6. **Missing libraries or headers**
   - Install development packages: `sudo apt install libc6-dev`
   - For specific libraries, install corresponding `-dev` packages

### Useful VS Code Shortcuts (Linux)

- `Ctrl+Shift+B`: Run build task
- `F5`: Start debugging
- `Ctrl+F5`: Run without debugging
- `Ctrl+`` ` `: Open integrated terminal
- `Ctrl+Shift+P`: Open Command Palette

## Getting Started

1. Clone or download this repository
2. Install build tools: `sudo apt update && sudo apt install build-essential`
3. Install Visual Studio Code
4. Install the required VS Code extensions
5. Create a Linux-specific `.vscode/tasks.json` file (see example above)
6. Open the project folder in VS Code
7. Open `main.cpp` and press `Ctrl+Shift+B` to build
8. Run the generated executable: `./main`

## Linux-Specific Notes

- Executables don't have `.exe` extensions on Linux
- Use `./` prefix to run executables (e.g., `./main` instead of `main.exe`)
- Linux uses `g++` as the standard C++ compiler
- Package management via `apt` (Ubuntu/Debian)
- File paths use forward slashes `/` instead of backslashes `\`
- Case-sensitive file system
- Standard compiler flags: `-Wall -Wextra -std=c++17`
