# CPP-VSC-Sample (macOS)

A sample C++ project configured for Visual Studio Code with build tasks and debugging support on macOS.

## Project Structure

```
CPP-VSC-Sample/
├── main.cpp            # Main C++ source file
├── VoltageCalc.cpp     # Voltage calculation program
...                     # Add your own cpp files
├── README.md           # Windows-specific README
├── README_MacOS.md     # This file (macOS-specific instructions)
└── .vscode/            # VS Code configuration files
    └── tasks.json      # Build tasks configuration (needs macOS adaptation)
```

## Prerequisites

### Required Software
- **Xcode Command Line Tools**: C++ compiler toolchain
  - Installation: `xcode-select --install`
  - Compiler path: `/usr/bin/clang++` or `/usr/bin/g++`
- **Homebrew** (Optional but recommended): Package manager for macOS
  - Installation: `/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"`
- **Visual Studio Code**: Code editor
  - Download from: [VS Code Official Website](https://code.visualstudio.com/)
  - Or install via Homebrew: `brew install --cask visual-studio-code`

### Alternative Compiler Installation via Homebrew

```bash
# Install GCC compiler (alternative to clang)
brew install gcc

# The compiler will be available as gcc-13 (or latest version)
# Example: /opt/homebrew/bin/gcc-13
```

### Required VS Code Extensions

Install the following extensions for the best C++ development experience:

1. **C/C++ Extension Pack** (`ms-vscode.cpptools-extension-pack`)
   - Includes C/C++ IntelliSense, debugging, and code browsing
   - Essential for C++ development in VS Code

2. **C/C++** (`ms-vscode.cpptools`)
   - Core C++ language support
   - Syntax highlighting, IntelliSense, and debugging

3. **C/C++ Themes** (`ms-vscode.cpptools-themes`)
   - Enhanced syntax highlighting themes for C++

4. **CMake Tools** (`ms-vscode.cmake-tools`) - Optional
   - For projects using CMake build system

### Installation Commands

You can install the required extensions using VS Code's command palette or terminal assuming you have installed Visual Studio Code correctly:

```bash
# Install C/C++ Extension Pack (includes core C++ extension)
code --install-extension ms-vscode.cpptools-extension-pack

# Install additional extensions
code --install-extension ms-vscode.cpptools-themes
```

## Building and Running

### Method 1: Using VS Code Build Task (Recommended)

**Note**: You'll need to create a macOS-specific `.vscode/tasks.json` file or modify the existing one:

1. Create or modify `.vscode/tasks.json`:

```json
{
    "version": "2.0.0",
    "tasks": [
        {
            "type": "cppbuild",
            "label": "C/C++: clang++ build active file",
            "command": "/usr/bin/clang++",
            "args": [
                "-fcolor-diagnostics",
                "-fansi-escape-codes",
                "-g",
                "${file}",
                "-o",
                "${fileDirname}/${fileBasenameNoExtension}"
            ],
            "options": {
                "cwd": "${fileDirname}"
            },
            "group": {
                "kind": "build",
                "isDefault": true
            },
            "detail": "Task generated by Debugger for macOS."
        }
    ]
}
```

2. Open the project in VS Code
3. Open the C++ file you want to compile (e.g., `main.cpp`)
4. Use the build task:
   - Press `Cmd+Shift+P` to open Command Palette
   - Type "Tasks: Run Build Task" and select it
   - Choose "C/C++: clang++ build active file"

### Method 2: Using Terminal

Open the integrated terminal in VS Code (`Ctrl+`` ` `) and run:

```bash
# Compile main.cpp or the corresponding name of the cpp file (using clang++)
clang++ -fcolor-diagnostics -fansi-escape-codes -g main.cpp -o main

# Or using g++ if installed via Homebrew
g++ -fdiagnostics-color=always -g main.cpp -o main

# Run the executable
./main
```

```bash
# Compile VoltageCalc.cpp
clang++ -fcolor-diagnostics -fansi-escape-codes -g VoltageCalc.cpp -o VoltageCalc

# Run the executable
./VoltageCalc
```

## Configuration

### macOS-Specific Configuration

- **Compiler**: `/usr/bin/clang++` (default) or `/usr/bin/g++`
- **Flags for clang++**: 
  - `-fcolor-diagnostics`: Colored compiler output
  - `-fansi-escape-codes`: ANSI escape codes for terminal colors
  - `-g`: Include debugging information

- **Flags for g++**:
  - `-fdiagnostics-color=always`: Colored compiler output
  - `-g`: Include debugging information

### File Permissions

On macOS, compiled executables don't have `.exe` extensions and may need execute permissions:

```bash
# Make executable (usually automatic)
chmod +x main

# Run executable
./main
```

## Troubleshooting

### Common Issues

1. **Compiler not found**
   - Install Xcode Command Line Tools: `xcode-select --install`
   - Verify installation: `clang++ --version` or `g++ --version`
   - Check the compiler path in `.vscode/tasks.json`

2. **VS Code doesn't recognize C++ syntax**
   - Install the C/C++ extension pack
   - Reload VS Code window (`Cmd+R`)

3. **Build task not available**
   - Ensure you have a C++ file open in the editor
   - Check that `.vscode/tasks.json` exists and is properly configured for macOS
   - Create a macOS-specific tasks.json file

4. **Permission denied when running executable**
   - Make sure the file has execute permissions: `chmod +x filename`

5. **Code command not found**
   - Install VS Code command line tools: Open VS Code → `Cmd+Shift+P` → "Shell Command: Install 'code' command in PATH"

### Useful VS Code Shortcuts (macOS)

- `Cmd+Shift+B`: Run build task
- `F5`: Start debugging
- `Cmd+F5`: Run without debugging
- `Ctrl+`` ` `: Open integrated terminal
- `Cmd+Shift+P`: Open Command Palette

## Getting Started

1. Clone or download this repository
2. Install Xcode Command Line Tools: `xcode-select --install`
3. Install the required VS Code extensions
4. Create a macOS-specific `.vscode/tasks.json` file (see example above)
5. Open the project folder in VS Code
6. Open `main.cpp` and press `Cmd+Shift+B` to build
7. Run the generated executable: `./main`

## macOS-Specific Notes

- Executables don't have `.exe` extensions on macOS
- Use `./` prefix to run executables (e.g., `./main` instead of `main.exe`)
- macOS uses `clang++` as the default C++ compiler
- Terminal shortcuts use `Cmd` instead of `Ctrl` for most operations
- File paths use forward slashes `/` instead of backslashes `\`
